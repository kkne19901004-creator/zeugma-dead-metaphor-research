# Codex Task Guide

Codexは、このリポジトリで研究補助者として作業する。目的は、くびき語法による dead metaphor の再活性化という仮説を、発表・修論に発展できる形へ整理することである。

## 基本方針

- Shen 1998を主軸にする。
- Goossens 1990は metaphtonymy の補助理論として扱う。
- くびき語法をmetaphtonymyと同一視しない。
- 読者の心理処理を断定しない。
- 「再解釈」よりも「再前景化」「再具体化」「比喩性の可視化」を優先する。
- 危険な主張は「危険」と明記する。

## 今後実行する作業

1. 文献リスト内外から関連研究を探す。
   - dead metaphor
   - conventional metaphor
   - metaphor revitalization
   - idiom literalization
   - idiom variation
   - defamiliarization
   - foregrounding
   - zeugma / syllepsis

2. dead metaphor 再活性化の理論的根拠を整理する。
   - dead metaphorとは何か。
   - conventional metaphorとの違いは何か。
   - 再活性化、再前景化、literalizationは先行研究でどう扱われているか。

3. くびき語法の非典型項分類を作る。
   - dead metaphor再活性化型
   - metaphor型
   - metonymy混合型
   - idiom型
   - polysemy型
   - frame shift型
   - 抽象化型

4. 英語例と日本語例を集める。
   - CSVに出典欄が必要なら追加する。
   - 日本語例は作例か実例かを必ず区別する。
   - 実例はコーパス・書籍・記事・広告などで出典を確認する。

5. 語順の入れ替えによる印象差を整理する。
   - 字義的項→比喩的項
   - 比喩的項→字義的項
   - 自然さ、くびき語法らしさ、再前景化の強さを分けて記録する。

6. 意味的距離の観点を整理する。
   - 距離が近い
   - 距離が適度
   - 距離が遠すぎる

7. 5枚程度の発表資料案を作る。
   - 問題設定
   - 先行研究
   - 中心仮説
   - 分析観点
   - 予備分析と今後の課題

8. 危険な主張と安全な主張を分ける。
   - 強く言えること
   - 仮説として言えること
   - まだ言わない方がよいこと

## Codexが出力するときの注意

- 論文に書かれていないことは「発展仮説」と明記する。
- 例文が作例の場合は「作例」と書く。
- くびき語法とdead metaphorの接続は、予備分析として扱う。
- 修論化する場合に必要な追加文献・追加データを必ず示す。
